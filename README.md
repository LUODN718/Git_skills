# Git Daily — Git 每日助手

给 Codex Agent 用的 Git 技能包。用日常对话完成 Git 操作，像和同事说话一样下指令。

## 安装

将 `git-daily/` 放入 `~/.codex/skills/`，重启 Codex。

## 使用

**显式**：`$git-daily 帮我提交今天的代码`

**自然**：只要话里带 Git 相关词（提交/push/分支/合并/冲突...）就会自动激活。

## 覆盖场景

| 场景 | 你说 |
|------|------|
| 🚀 初始化绑定 | 我刚建了 GitHub 仓库，帮我绑一下 |
| 💾 日常提交 | 帮我把今天写的提交一下 |
| 🔄 协作同步 | 检查一下远程有什么更新 |
| 👀 看同事工作 | 看看小王昨天在 feature/report 上写了什么 |
| 🔀 智能合并 | 帮我把 feature/login 合并到 main |
| 🔍 代码审查 | 看看这个分支的改动，给个审查意见 |
| 🌿 分支管理 | 新建 feature/xxx、切分支、删分支 |
| ⚡ 合并冲突 | 有冲突，帮我看看怎么解决 |
| ⏪ 回退撤销 | 刚才提交不对，帮我撤销但保留改动 |
| 📦 暂存切换 | 先暂存，切去修个 bug，回来继续 |
| 📝 README 生成 | 帮我总结项目，写个 README |
| 📜 查看历史 | 看看日志 / 这个文件谁改的 |
| 🏷️ 标签管理 | 打 v1.0 标签、推送标签 |
| 🍒 cherry-pick | 把那个提交摘过来 |
| 🙈 gitignore | 帮我生成 .gitignore |

## 智能合并特色

合并前自动分析：
- 两个分支各有多少独有提交
- 文件变更统计、提交列表
- 预判是否有冲突

有冲突时：
- 列出冲突文件，展示冲突内容
- 分析每个冲突原因，给出"保留谁的"建议
- 可代为解决冲突并完成合并

无冲突时：
- 按改动量给出合并建议（直接合 / review 再合 / 走 PR）
- 一键执行合并

## 设计原则

1. 口语优先 — 像跟同事说话
2. 先看后动 — 修改前必查状态
3. 安全默认 — 不擅自 --force、--hard、删未合并分支
4. 主动确认 — 缺信息用自然语言问
5. 合并建议 — 分析改动量和风险再建议
